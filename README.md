# OpenInvestigator

Case intelligence system for hypothesis-driven analysis of complex data.

## 🧠 Vision

OpenInvestigator is designed to support human reasoning in complex investigations and analytical workflows.

Instead of only processing data, it helps users:

- explore multiple hypotheses
- structure incomplete and conflicting information
- build transparent, reviewable case understanding

The goal is not decision automation, but better thinking made explicit and reproducible.

For a short note on where the project comes from, see [`docs/vision.md`](docs/vision.md).

## 🚀 Motivation

OpenInvestigator grew out of a simple observation:

many tools help store, search, and summarize information — but very few help people systematically build, test, and compare hypotheses in complex cases.

This project is an attempt to close that gap.

## ❗ The problem

Complex investigations — whether in journalism, OSINT, security, or law enforcement — tend to share the same challenges:

- fragmented and contradictory information
- implicit, untracked hypotheses
- analysis that is hard to reproduce
- tools optimized for storage or retrieval, not reasoning

As a result, important insights are often missed — not because of lack of skill, but because of lack of structure.

## 💡 What makes this different

OpenInvestigator is built around a hypothesis-driven workflow:

- hypotheses as first-class artifacts
- source-grounded reasoning
- multiple investigation runs on the same case data
- comparison of competing analytical perspectives

This shifts the focus from:

> What data do we have?  
> to  
> What could be true — and how do we test it?

## 🏗️ Architecture

The current system concept is centered on:

- stable case data
- reproducible investigation runs
- deliberate comparison across runs

See [`docs/architecture.md`](docs/architecture.md) for the current architecture and MVP direction.

## 🔧 Status

Early-stage project.  
Currently focused on core architecture and first MVP scope.

## 🤝 Principles

- human-in-the-loop by design
- transparency over black-box outputs
- local-first where possible
- structured artifacts over free-form analysis
- provenance as a core requirement
